# Windows 客户端设计说明

新版客户端采用亮色安全工作台风格，面向“HGCN 深度漏洞检测 + MEDE 静态证据验证”的完整流程。

## 页面结构

1. 总览：展示 HGCN/MEDE 工作流、核心指标和当前配置。
2. 任务配置：配置仓库、数据集或 HCPG pkl、HGCN checkpoint、MEDE、fallback、可选 CPG/HCPG 工件。
3. 检测流程：以流水线卡片展示 CPG/HCPG 可选步骤、HGCN 检测、MEDE 和交叉验证。
4. 结果分析：展示预测结果、静态证据和交叉验证后的告警列表。
5. 报告导出：导出 HTML 报告并打开输出目录。

## 设计原则

- HGCN 是当前深度检测主入口。
- MEDE 提供静态证据，用于交叉验证和告警降噪。
- UI 只负责任务编排、过程展示和结果导出，不重写核心算法。
- fallback 仅用于流程演示，并在输出结果中明确标记。
